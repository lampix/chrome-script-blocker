Script Blocker — Options

Block individual scripts per domain. Each rule consists of a domain (on which sites the rule applies) and a URL pattern (which script to block). Leave the domain empty for a global rule.

On Domain Script URL pattern
URL pattern syntax

Patterns follow Chrome's declarativeNetRequest.urlFilter syntax, similar to uBlock/AdBlock Plus:

  • ||example.com/path — matches example.com and all subdomains, starting at the path
  • * — wildcard (any characters)
  • ^ — separator (/, ?, =, &, or end of URL)
  • | at start/end — anchored to the start/end of the URL

Examples:

  • ||cdn.example.com/tracker-*.js — tracker script with any hash suffix
  • ||analytics.example.com^ — entire domain
  • /tracker.js — any URL containing "/tracker.js"
